║                          🔍 VÉRIFICATIONS                                  ║
║                                                                            ║
╚════════════════════════════════════════════════════════════════════════════╝

  1. Trigger actif ?
     
     SELECT tgname FROM pg_trigger 
     WHERE tgname = 'notify_client_on_mission_accepted';
     
     → Doit retourner 1 ligne

  2. Notification créée après acceptation ?
     
     SELECT * FROM notifications 
     WHERE type = 'mission_accepted' 
     ORDER BY created_at DESC LIMIT 5;
     
     → Doit montrer les notifications récentes

  3. Aucune mission sans notification ?
     
     SELECT COUNT(*) FROM missions m
     LEFT JOIN notifications n ON m.id = n.mission_id 
       AND n.type = 'mission_accepted'
     WHERE m.status = 'accepted' 
       AND m.accepted_at > NOW() - INTERVAL '24h'
       AND n.id IS NULL;
     
     → Doit retourner 0

  • Le trigger n'apparaît pas ?

  • La notification n'est pas créée ?
    → Vérifier que le trigger est actif (requête ci-dessus)
    → Consulter les logs Supabase (Database > Logs)
