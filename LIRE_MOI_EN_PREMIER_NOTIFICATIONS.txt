📋 PROBLÈME
───────────
Les clients ne reçoivent PAS de notification quand un artisan accepte 
leur mission, bien que l'interface dise "Le client a été notifié".


⏱️  TEMPS DE CORRECTION : 5 minutes


🚀 ACTIONS À FAIRE MAINTENANT
──────────────────────────────

1. Ouvrir Supabase SQL Editor
   → https://supabase.com
   → Sélectionner votre projet
   → Cliquer "SQL Editor" → "New query"

2. Copier-coller le script
   → Ouvrir : COPIER_COLLER_SUPABASE_NOTIFICATIONS.sql
   → Sélectionner TOUT (Ctrl+A)
   → Copier (Ctrl+C)
   → Coller dans Supabase (Ctrl+V)

3. Exécuter
   → Cliquer "Run" (ou Ctrl+Enter)
   → Attendre 2 secondes
   → Voir : ✅ CORRECTION APPLIQUÉE AVEC SUCCÈS !

4. Tester
   → App : Accepter une mission en tant qu'artisan
   → Supabase : Exécuter cette requête
   
     SELECT * FROM notifications 
     WHERE type = 'mission_accepted' 
     ORDER BY created_at DESC LIMIT 5;
   
   → ✅ Notification doit apparaître


✅ RÉSULTAT
────────────
Après cette correction, 100% des missions acceptées génèrent 
automatiquement une notification pour le client.

Garanti par le database. Plus d'erreurs silencieuses.
